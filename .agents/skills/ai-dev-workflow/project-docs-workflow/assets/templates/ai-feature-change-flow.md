# AI 功能变更流程

## 目标

让 AI 在开发前读取最小上下文，开发后更新相关文档，并给出可验证交付结果。

## 开发前

1. 读取 `docs/index.md`。
2. 根据任务类型读取相关 `features/`、`api/`、`data/`、`tests/`、`architecture/`、`flows/` 或 `adr/`。
3. 对照代码确认文档是否过期。
4. 若发现文档 drift，先标记再继续。

## 开发中

1. 按项目现有结构修改代码。
2. 记录影响到的功能、接口、数据、测试、流程或架构文档。
3. 重要技术取舍需要补 ADR。

## 开发后

1. 更新相关 docs 文件。
2. 更新 `docs/index.md` 的模块索引和状态。
3. 判断 README 是否需要更新。
4. 运行或说明验证命令。
5. 给出中文 Conventional Commit 建议。

## 不做

- 不把整个 docs 目录全部塞进上下文。
- 不把候选规则直接写进 `AGENTS.md`。
- 不声称未运行的测试已通过。
