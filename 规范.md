# PowerShell Profile 规范

## `profile.ps1` 函数命名

- `getskill` 是唯一允许不使用 `GS-` 前缀的公开入口函数。
- 单个 skill、多个 skill、配套 skill set 都必须通过 `getskill` 暴露，不新增 `getskills`、别名或其他公开入口。
- `profile.ps1` 中其余函数必须全部使用 `GS-` 前缀，避免安装器在清理旧函数时误删用户自定义函数。

## `profile.ps1` 结构约束

- 所有函数必须是扁平的顶层函数定义。
- 不允许在函数内部再定义函数。
- 不允许通过 `Set-Alias` 或 `New-Alias` 暴露命令入口。

## 安装器同步规则

- `src/install.ps1` 必须从 `profile.ps1` 模板中自动提取函数名。
- 安装器在写入 managed block 前，必须先删除用户 profile 中同名的旧模板函数定义。
- 新增、删除或重命名 `profile.ps1` 函数时，优先保持安装器基于模板自动同步，不要维护手写函数名列表。
